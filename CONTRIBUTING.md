## Code of Conduct

All contributors are expected to follow our [Code of Conduct](CODE_OF_CONDUCT.md) to ensure a welcoming environment for everyone.

## How to Contribute

1. **Fork** the repository.
2. **Clone** your fork to your local machine:
   ```
   git clone https://github.com/your-username/TLPH.git
   ```
3. **Create a branch** for your feature or fix:
   ```
   git checkout -b your-branch-name
   ```
4. **Make your changes** and **commit** them with clear, descriptive messages.
5. **Push** your branch to GitHub:
   ```
   git push origin your-branch-name
   ```
6. **Open a Pull Request** with a clear description of your changes.

## Pull Requests

- Use concise and descriptive titles.
- Provide context and a detailed description of your changes.
- Link related issues (e.g., `Fixes #XX`).
- Ensure your branch is up to date with `main`.
- Make sure all checks pass (tests, linters, etc).

## Coding Guidelines

- Follow PEP8 guidelines for Python code.
- For HTML, CSS, and JavaScript, use consistent formatting and indentation.
- For C and PowerShell, follow language-specific best practices.
- Add clear comments and docstrings where appropriate.
- Write tests for any new features or fixes.

## Bug Reports

- Use [GitHub Issues](https://github.com/alqzdave/TLPH/issues) to report bugs.
- Include OS, Python version, and relevant logs/error messages.
- Describe steps to reproduce the bug.

## Feature Requests

- Use [GitHub Issues](https://github.com/alqzdave/TLPH/issues) to suggest features.
- Explain the motivation and expected outcomes.

## Documentation

- Check for typos, grammar, and formatting.
- Improve code and docstrings where possible.
- Update the README as needed.

## Community & Support

For questions or support, open an issue or start a discussion on GitHub.
